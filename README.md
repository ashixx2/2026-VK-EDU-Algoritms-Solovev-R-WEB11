## Описание решения задач в модуле 2

### Задача 1. Хеш-таблица

Реализовано множество строк на основе динамической хеш-таблицы с открытой адресацией.

Вариант `1_1`: для разрешения коллизий используется квадратичное пробирование.

### Задача 2. Порядок обхода бинарного дерева

Построено бинарное дерево поиска наивным порядком вставки.

Вариант `2_2`: вывод элементов выполняется в порядке `pre-order`.

### Задача 3. B-дерево

Вариант `4_1`: добавление солдата и удаление по позиции выполняются за `O(log n)`.

## Описание решения задач в модуле 3

### Задача 1. Представление графа

Реализованы четыре способа хранения ориентированного графа через общий интерфейс `IGraph`.

Реализованные представления:

* `ListGraph` — массив списков смежности;
* `MatrixGraph` — матрица смежности;
* `SetGraph` — массив множеств;
* `ArcGraph` — массив пар ребер.

Каждая реализация поддерживает добавление ребра, получение количества вершин, получение исходящих и входящих вершин.

Также реализовано копирование графа из объекта, поддерживающего интерфейс `IGraph`.

### Задача 2. Количество различных кратчайших путей

Реализован подсчет количества различных кратчайших путей между двумя вершинами в невзвешенном неориентированном графе.

Для решения используется обход в ширину `BFS`.

Во время обхода для каждой вершины хранятся:

* длина кратчайшего пути от стартовой вершины;
* количество кратчайших путей до этой вершины.

Сложность решения:

```text
O(V + E)
```

### Задача 3. Города

Реализован поиск самого выгодного маршрута между двумя городами во взвешенном неориентированном графе.

Города представлены вершинами графа, дороги — ребрами, время в пути — весами ребер.

Для решения используется алгоритм Дейкстры с очередью с приоритетом.

Сложность решения:

```text
O((N + M)logN)
```
