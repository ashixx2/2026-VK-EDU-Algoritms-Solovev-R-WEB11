# Задача 1. Представление графа

## Описание

Реализованы четыре способа хранения ориентированного графа через общий интерфейс `IGraph`.

По условию требуется создать следующие реализации:

- `ListGraph` — хранение графа в виде массива списков смежности;
- `MatrixGraph` — хранение графа в виде матрицы смежности;
- `SetGraph` — хранение графа в виде массива множеств;
- `ArcGraph` — хранение графа в виде массива пар ребер.

Все классы поддерживают одинаковый набор операций:

- добавление ориентированного ребра;
- получение количества вершин;
- получение исходящих вершин;
- получение входящих вершин;
- создание копии графа из объекта, реализующего интерфейс `IGraph`.

## Интерфейс графа

Общий интерфейс задает методы, которые должны быть реализованы во всех классах.

```cpp
struct IGraph {
    virtual ~IGraph() {}

    virtual void AddEdge(int from, int to) = 0;
    virtual int VerticesCount() const = 0;

    virtual std::vector<int> GetNextVertices(int vertex) const = 0;
    virtual std::vector<int> GetPrevVertices(int vertex) const = 0;
};
```

Благодаря общему интерфейсу с любым представлением графа можно работать одинаково, не обращаясь напрямую к его внутренней структуре данных.

## Логика работы реализаций

`ListGraph` хранит для каждой вершины список вершин, в которые из нее выходят ребра. Такое представление удобно для разреженных графов, так как в памяти находятся только существующие ребра.

`MatrixGraph` хранит информацию о ребрах в матрице смежности. Наличие ребра определяется значением в ячейке матрицы. Такой вариант требует больше памяти, но напрямую хранит связь между любой парой вершин.

`SetGraph` хранит для каждой вершины множество исходящих вершин. В отличие от списка смежности, множество не допускает повторяющихся ребер и поддерживает упорядоченное хранение элементов.

`ArcGraph` хранит граф как общий массив ребер. Каждое ребро представлено парой вершин `{from, to}`. Для получения соседей вершины выполняется проход по массиву ребер.

## Копирование графа

Каждая реализация имеет конструктор, принимающий объект типа `const IGraph&`.

Это позволяет создать граф одного типа на основе графа другого типа. Копирование выполняется через общий интерфейс: для каждой вершины запрашиваются исходящие вершины, после чего соответствующие ребра добавляются в новый граф.

Общая логика копирования:

```cpp
for (int from = 0; from < graph.VerticesCount(); ++from) {
    std::vector<int> nextVertices = graph.GetNextVertices(from);

    for (int to : nextVertices) {
        AddEdge(from, to);
    }
}
```

Такой подход позволяет не зависеть от внутреннего устройства исходного графа.
