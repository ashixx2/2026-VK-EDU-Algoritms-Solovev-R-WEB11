# Задача 2. Порядок обхода

## Вариант `2_2`.

Необходимо вывести элементы бинарного дерева в порядке `pre-order`.

Порядок обхода `pre-order`:

```text
корень → левое поддерево → правое поддерево
```

## Условие

Дано число `N` и последовательность целых чисел длиной `N`.

Необходимо построить бинарное дерево поиска, заданное наивным порядком вставки.

При добавлении очередного числа `K`:

* если `K` меньше значения в текущем узле, переход выполняется в левое поддерево;
* иначе переход выполняется в правое поддерево.

После построения дерева нужно вывести его элементы в порядке `pre-order`.

## Требования

Рекурсия не используется.
Дерево реализовано в виде класса.
Поддерживается передача функции сравнения снаружи.

## Реализация

Дерево реализовано шаблонным классом `BinarySearchTree`.

Каждый узел дерева хранит:

значение `key`;
индекс левого потомка;
индекс правого потомка.

Узлы хранятся в массиве `std::vector`.

Вместо указателей используются индексы узлов. Если потомка нет, хранится значение `-1`.

## Логика работы программы

Сначала программа считывает число `N`.

Затем считываются `N` чисел. Каждое число сразу добавляется в бинарное дерево поиска.

При вставке программа начинает с корня дерева и двигается вниз:

* если новое значение меньше значения в текущем узле, выполняется переход влево;
* иначе выполняется переход вправо.

Если нужного потомка нет, создаётся новый узел.

После построения дерева выполняется обход `pre-order`.

Обход реализован с помощью стека.

В стек сначала помещается корень дерева.

Пока стек не пустой:

1. Из стека достаётся текущий узел.
2. Значение узла выводится.
3. В стек добавляется правый потомок.
4. В стек добавляется левый потомок.

Правый потомок добавляется раньше левого, потому что стек работает по принципу LIFO. Благодаря этому левый потомок будет обработан раньше правого.

## Сложность

Средняя сложность построения дерева:

```text
O(n log n)
```

В худшем случае, если дерево вырождается в цепочку:

```text
O(n^2)
```

Сложность обхода дерева:

```text
O(n)
```

Дополнительная память для обхода:

```text
O(n)
```

## Пример

Ввод:

```text
4
3 1 4 2
```

Построенное дерево:

```text
    3
   / \
  1   4
   \
    2
```

Вывод `pre-order`:

```text
3 1 2 4
```
