# Задача 4. Солдаты

## Вариант `4_1`.

## Условие

Необходимо поддерживать строй солдат, отсортированный по росту.

Солдаты должны стоять от самого высокого к самому низкому.

Есть два типа команд:

`1 X` — солдат ростом `X` приходит в строй;
`2 Y` — нужно удалить солдата, стоящего на позиции `Y`.

Позиции в строю нумеруются с нуля.

Для каждой команды добавления нужно вывести позицию, на которую должен встать новый солдат.

## Требования

Скорость выполнения каждой команды — `O(log n)`.
Используется АВЛ-дерево.
Решение поддерживает передачу функции сравнения снаружи.

## Используемая структура данных

Для решения используется АВЛ-дерево с хранением размера поддерева в каждой вершине.

Каждая вершина дерева хранит:

`key` — рост солдата;
`height` — высоту вершины;
`size` — количество вершин в поддереве;
`left` — указатель на левое поддерево;
`right` — указатель на правое поддерево.

Поле `size` нужно для быстрого определения позиции солдата и удаления по номеру позиции.

## Порядок сортировки

В строю солдаты стоят по убыванию роста.

Для этого используется компаратор:

```cpp
struct Comparator {
    bool operator()(int left, int right) const {
        return left > right;
    }
};
```

Если `left > right`, значит солдат с ростом `left` должен стоять раньше солдата с ростом `right`.

## Логика добавления солдата

При добавлении нового солдата дерево ищет место для его роста.

Если новый рост больше текущего, переход выполняется в левое поддерево.

Если новый рост меньше текущего, переход выполняется в правое поддерево.

Во время спуска по дереву считается позиция нового солдата.

Когда новый солдат уходит вправо, это значит, что перед ним уже стоят:

* все солдаты из левого поддерева текущей вершины;
* сам солдат в текущей вершине.

Поэтому к позиции прибавляется:

```text
size(left) + 1
```

После вставки вершины дерево балансируется с помощью поворотов.

## Логика удаления солдата

Удаление выполняется не по росту, а по позиции в строю.

Для этого используется поле `size`.

В каждой вершине считается количество солдат в левом поддереве:

```text
left_size = size(left)
```

Дальше возможны три случая:

1. Если удаляемая позиция меньше `left_size`, нужно идти в левое поддерево.
2. Если удаляемая позиция больше `left_size`, нужно идти в правое поддерево.
3. Если позиция равна `left_size`, текущая вершина соответствует удаляемому солдату.

После удаления дерево также балансируется.

## Балансировка

АВЛ-дерево поддерживает разницу высот левого и правого поддерева не больше одного.

После каждой вставки и удаления обновляются:

* высота вершины;
* размер поддерева.

Если баланс нарушен, выполняется один из поворотов:

* левый поворот;
* правый поворот;
* большой левый поворот;
* большой правый поворот.

## Сложность

Так как используется АВЛ-дерево, высота дерева равна:

```text
O(log n)
```

Добавление солдата выполняется за:

```text
O(log n)
```

Удаление солдата по позиции выполняется за:

```text
O(log n)
```

## Пример

Ввод:

```text
5
1 100
1 200
1 50
2 1
1 150
```

Вывод:

```text
0
0
2
1
```
