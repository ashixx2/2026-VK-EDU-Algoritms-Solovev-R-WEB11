# Задача 1. Хеш-таблица

## Вариант 1_1

Для разрешения коллизий используется квадратичное пробирование.

## Условие

Необходимо реализовать множество строк на основе динамической хеш-таблицы с открытой адресацией.

Хранимые строки непустые и состоят из строчных латинских букв.

Программа должна поддерживать три операции:

1) `+ string` — добавление строки в множество;
2) `- string` — удаление строки из множества;
3) `? string` — проверка принадлежности строки множеству.

Для каждой операции программа выводит:

`OK`, если операция выполнена успешно;
`FAIL`, если операция не может быть выполнена.

## Используемая структура данных

Хеш-таблица реализована в виде класса `HashTable`.

Таблица представляет собой массив ячеек. Каждая ячейка хранит строку и состояние.

Состояния ячейки:

`EMPTY` — ячейка пустая и раньше не использовалась;
`FILLED` — в ячейке хранится строка;    
`DELETED` — строка была удалена, но ячейка раньше использовалась.

Состояние `DELETED` необходимо, чтобы правильно выполнять поиск после удаления элементов. Если после удаления делать ячейку полностью пустой, поиск может остановиться раньше времени и не найти элемент, который был добавлен дальше из-за коллизии.

## Хеш-функция

Для строки вычисляется хеш с помощью метода Горнера.

Строка обрабатывается посимвольно. На каждом шаге текущее значение хеша умножается на основание и к нему прибавляется код очередного символа.

В программе используется основание:

```cpp
const size_t base = 31;
```

После вычисления хеша начальная позиция в таблице определяется так:

```cpp
index = hash % table.size();
```

## Логика работы программы

Программа считывает операции до конца входного потока.

Для каждой строки входных данных считываются:

```text
operation string
```

где `operation` — один из символов `+`, `-`, `?`, а `string` — строка, с которой выполняется операция.

Потом программа выбирает нужное действие:

```cpp
if (operation == '+') {
    result = hash_table.add(str);
} else if (operation == '-') {
    result = hash_table.remove(str);
} else if (operation == '?') {
    result = hash_table.contains(str);
}
```

После выполнения операции результат выводится в стандартный поток вывода:

* если метод вернул `true`, выводится `OK`;
* если метод вернул `false`, выводится `FAIL`.

## Добавление элемента

При добавлении строки сначала проверяется коэффициент заполнения таблицы.

Если количество занятых ячеек достигло `3/4` от размера таблицы, выполняется перехеширование.

После этого вычисляется хеш строки и начальная позиция в таблице.

Если ячейка свободна, строка добавляется в неё.

Если ячейка занята другой строкой, применяется квадратичное пробирование:

```text
h, h + 1, h + 1 + 2, h + 1 + 2 + 3, ...
```

В программе переход к следующей позиции выполняется так:

```cpp
index = (index + i + 1) % table.size();
```

Если при поиске места встречается такая же строка, добавление не выполняется, потому что множество не должно хранить одинаковые строки.

Если во время поиска встретилась удалённая ячейка, программа запоминает её как возможное место для вставки.

## Удаление элемента

При удалении строки программа вычисляет её хеш и начинает поиск с соответствующей позиции.

Если строка найдена, ячейка помечается как `DELETED`.

Если во время поиска встречается ячейка `EMPTY`, значит строка отсутствует в таблице, и удаление завершается неудачно.

## Проверка наличия элемента

Проверка наличия строки выполняется аналогично удалению.

Программа вычисляет хеш строки, получает начальную позицию и идёт по последовательности проб.

Если строка найдена, возвращается `true`.

Если встречается ячейка `EMPTY`, поиск прекращается, потому что дальше по этой последовательности проб нужного элемента быть не может.

Если встречается ячейка `DELETED`, поиск продолжается.

## Перехеширование

Перехеширование выполняется при достижении коэффициента заполнения `3/4`.

При перехешировании размер таблицы увеличивается в два раза.

После этого все существующие элементы заново добавляются в новую таблицу.

Удалённые ячейки при перехешировании не переносятся.

Это позволяет уменьшить количество ячеек со состоянием `DELETED` и ускорить дальнейшие операции.

## Сложность

Средняя сложность операций добавления, удаления и поиска:

```text
O(1)
```

В худшем случае сложность одной операции может достигать:

```text
O(n)
```

Перехеширование выполняется за:

```text
O(n)
```

## Пример

Ввод:

```text
+ hello
+ bye
? bye
+ bye
- bye
? bye
? hello
```

Вывод:

```text
OK
OK
OK
FAIL
OK
FAIL
OK
```
